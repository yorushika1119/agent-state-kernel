提交：

```text

```

变化：

- 新增 `route_task_context_with_llm()`。

接受策略核心：

| 条件 | 结果 |
|---|---|
| LLM confidence 低于阈值 | 拒绝，回退规则 |
| target_task_id 不在候选 task 中 | 拒绝 |
| LLM 和规则目标一致 | 接受 |
| 规则分数支持 LLM 目标 | 接受 |
| 规则要求澄清但 LLM 高置信消歧 | 接受 |
| LLM 建议澄清 | 可较宽松接受，因为不会打断任务 |

```text
结论：暂不切换。
```

原因：

- 当前 reducer、pipeline、view 和测试仍以 `intent_states / plan_states / belief_items / commitments` 为事实来源。
- `task_brief_states / task_flows / claim_items / todo_obligations` 仍是兼容影子层。
- 现在硬切会影响打断、恢复、direct responder、progress synthesize 等稳定链路。

改进方案：

1. 继续保持旧表为事实来源。
2. 新表继续作为 task-first 读模型和兼容层。
3. 等 task-local task_flow / claim / todo 写入路径完全稳定后，再逐表切换读取优先级。
4. 每切一张表都必须有回归测试和 rebuild 测试。

当前边界：

- SSE 当前是轻量轮询 stream，不是独立后台 push broker。
- Notification 去重/节流是第一版策略，只处理同 task 同类型通知。
- RuntimeEventAdapter 是通用 HTTP client，不接管 runtime session DB 或 message history。
