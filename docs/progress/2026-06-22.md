### 3. LLM Task Router fallback

提交：

```text
6d640f0 Add optional LLM task routing fallback
ed282ef Add route acceptance policy
```

变化：

- 新增 `route_task_context_with_llm()`。
- 默认不调用真实模型，需要开启 `KMS_ENABLE_LLM_ROUTER=1` 或显式启用。
- LLM 只在规则低置信或需要澄清时参与。
- LLM 只能输出结构化 route decision，不直接决定是否打断或执行。
- 新增接受策略，不再只相信 LLM 自报 confidence。

接受策略核心：

| 条件 | 结果 |
|---|---|
| LLM confidence 低于阈值 | 拒绝，回退规则 |
| target_task_id 不在候选 task 中 | 拒绝 |
| LLM 和规则目标一致 | 接受 |
| 规则分数支持 LLM 目标 | 接受 |
| 规则要求澄清但 LLM 高置信消歧 | 接受 |
| LLM 建议澄清 | 可较宽松接受，因为不会打断任务 |

新增 API：

```text

```

变化：

- `observer_view` 提供安全进度、可转述摘要、safe facts、uncertain points、open todos、blocking reason、允许/禁止动作、pending observer notifications。

- `manager_view` 用于用户管理界面或没有 Talker 时的直接状态查看。
